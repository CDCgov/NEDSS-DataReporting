# Comparison Fixtures (APP-471)

Synthetic NBS_ODSE INSERTs designed to maximize RDB_MODERN coverage when RTR
runs initial hydration. Output is consumed by a planned RDB-vs-RDB_MODERN
comparison test against MasterETL.

## Status — 2026-05-21

End-to-end `scripts/merge_and_verify.sh` against 118 in-scope RDB_MODERN
target tables:

- **Fully covered: 65** (every column populated for at least one row)
- **Partially covered: 32**
- **Empty: 20**
- **Missing from live schema: 1** (`job_batch_rebuild_log` — not in baseline 6.0.18.1)
- **Overall column coverage: 33.9%** (1566 / 4615 columns)
- **10 RTR bugs surfaced and documented** (5 merged upstream, 3 squashed
  on this branch, 2 documented with repros for follow-up)

Wall-clock for the full pipeline: ~5 min from `docker compose down -v`.
Zero external dependencies beyond the baseline image.

## Build order

See `STRATEGY.md` → "Build order". Summary:

1. **Phase 0** (`prompts/phase_0_target_columns.md`) → `catalog/rtr_target_columns.md`
2. **Phase B** (`prompts/phase_b_edge_catalog.md`) → `catalog/edge_types.md`
3. **Tier 0** (`prompts/tier_0_foundation.md`) → `fixtures/00_foundation/`
4. **Provider canary** (`prompts/tier_1_provider_canary.md`) → derives Tier 1 template
5. **Tier 1 batch** — per-subject (~11 agents)
6. **Tier 2** — cross-subject links (one agent per edge type)
7. **Tier 3** — gap-driven SP coverage (Phase-2 full-chain fixtures: TB, COVID, Var, STD/HIV, BMIRD, d_investigation_repeat)
8. **Final merge + verification** — `scripts/merge_and_verify.sh`

## Running the pipeline

```sh
cd utilities/comparison-fixtures
./scripts/merge_and_verify.sh     # ~5 min, runs the full 9-step Merge contract
./scripts/coverage_summary.sh     # Regenerates coverage_merged.md
head -12 coverage/coverage_merged.md   # Headline numbers
```
